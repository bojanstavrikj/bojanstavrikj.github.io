Projects

Developing New Control Tool - Citibank

I developed a new controlling tool for the team in excel. It spread over more than 10 interconnected sheets controlling and reconciling data from 3 different systems on several levels of grain. The challenge was to have all the data in one place while making it run fast without having to go through many formulas to obtain the final result. Data would be inputted over several sheets, from 3 different sources; manual ledger adjustments would be added to the file depending on where the system is breaking and needs to be adjusted. All of this would then be summarized as a simulation of what the final numbers would be after all necessary adjustments are uploaded in the system. This summary was shown on day-to-date, month-to-date and year-to-date basis, including a difference calculation to the initial risk system. After all of this was done, a final system number would be inputed for a final check. Once the trading desk approves and agrees with the final reported numbers, a macro was created in order to distribute the numbers to a list of managers within the company.

Demand Forecasting - NOVA IMS

This project was developed in a team of 4. The data used is real and obtained from a transportation company based in Algarve, Portugal whose name cannot be disclosed. The company had trouble in determining the right number of drivers to hire for different periods of the year. That is where my colleagues and I came in. We built a predictive model using Machine Learning techniques for estimating expected number of services. The challenges faced in this project were data quality and aggregation. On top of that the client asked for 4 week batch predictions 8 weeks in advance, due to long hiring processes. The client asked for a maximum mean absolute error of 10% in the model. We managed to achieve a 5% error on average.
